El pedido a proveedor sirve para convertir una necesidad en un compromiso real de compra.

## ¿Para qué sirve esta página?

Permite gestionar:

- el proveedor,
- las líneas pedidas,
- el estado del pedido,
- la fecha de entrega,
- las cantidades comprometidas para una oleada,
- el paso hacia la recepción y luego el stock.

## Lo que suele preguntar el usuario

### ¿Pedido significa necesidad cubierta?

No del todo. Un pedido ayuda al planning, pero la ejecución solo queda realmente segura cuando hay stock real y luego lotes asignados.

### ¿Por qué vincular un pedido a una oleada?

Para saber qué pedido sostiene qué plan de producción.

## Errores frecuentes

- pensar que “entregado” significa que el stock ya existe,
- comprometer una cantidad que no responde a la prioridad real,
- dejar incompletos los datos necesarios para pasar a stock controlado.
