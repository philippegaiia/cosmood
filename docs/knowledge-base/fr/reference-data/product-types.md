Le type de produit est le cadre operationnel du produit.

## Cette page sert a quoi ?

Elle definit notamment :

- le mode de calcul du batch,
- la taille de batch par defaut,
- les unites attendues,
- la perte estimee,
- le modele QC partage,
- les lignes de production autorisees,
- la ligne par defaut.

## Ce que l'utilisateur se demande souvent

### Pourquoi creer le type avant le produit ?

Parce que le produit herite de ce cadre pour le planning et une partie du comportement de production.

### A quoi servent les lignes autorisees ?

Elles evitent de planifier un type de produit sur une ligne non adaptee.

## Erreurs frequentes

- laisser un type trop generique,
- oublier la ligne par defaut,
- utiliser de mauvais presets de batch.
